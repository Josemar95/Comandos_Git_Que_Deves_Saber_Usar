COMANDOS GIT BÁSICOS MAIS USADOS POR MIM:

1) Git Clone

Esse comando Git cria uma cópia exata de um repositório já existente.

2) git merge branch (develop)

Esse código diz que, tudo que está na branch develop irá para tua branch;

3) git add

adiciona seus arquivos modificados.

4) git commit -m "comentário"

Faz o comentário de arquivos que foram adicionados;

5) git branch

5.1) git branch

(lista todas as ramificações);

5.2) git branch

(cria um branch com o nome especificado);

5.3) git branch -d

(deleta o branch com o nome especificado).

6) git checkout

Esse comando Git pode ser utilizado para trocar de uma ramificação para outra.

Exemplo: git checkout .

Também é possível combinar operações, criando e fazendo o checkout de uma nova branch com um único comando:

Exemplo: git checkout -b

COMANDOS GIT INTERMEDIÁRIO MAIS USADOS POR MIM:

1) git remote

O comando Git remote estabelece uma conexão entre seu repositório local e um repositório remoto.

Exemplo: git remote add (nome)

Crie uma nova conexão com um repositório remoto.
Depois de adicionar um controle remoto, você poderá usá-lo ＜name＞como um atalho conveniente para ＜url＞outros comandos Git.

1.1) git remote rm

Remova a conexão com o repositório remoto chamado ＜name＞.

1.2) git remote rename

Renomeie uma conexão remota de ＜old-name＞para ＜new-name＞.

2) git fetch

Quando você precisa baixar as mudanças criadas por outros membros do seu projeto colaborativo, você precisa do comando Git fetch.

A partir desse comando, você irá receber todas as informações de commits, para avaliar, antes de aplicar essas alterações na sua versão local do repositório.

2.1) git fetch

Pega todos os branches do repositório. Isso também baixa todos os commits e arquivos necessários de outro repositório.

2.2) git fetch

Igual ao comando acima, mas busca apenas o branch especificado.

2.3) git fetch --all

Um movimento de energia que busca todos os controles remotos registrados e suas ramificações.

2.4) git fetch --dry-run

A --dry-runopção executará uma demonstração do comando. Ele produzirá exemplos de ações que executará durante a busca, mas não os aplicará.

3) git stash

Esse comando Git armazena temporariamente seus arquivos modificados em uma área chamada stash (“esconderijo”), sem interagir com os outros arquivos até ser necessário.

3.1) git stash list

Para listar todos os seus “esconderijos”.

3.2) git stash apply

Quando for o momento de aplicar o conteúdo do stash a um branch, usamos o parâmetro “apply”.

4) git show

Quer detalhes específicos sobre um commit que o log não mostra? O comando Git show é a resposta.

5) git rm

Para remover arquivos da sua pasta, você pode utilizar o comando Git rm. Exemplo: $ git rm

COMANDOS GIT AVANÇADOS MAIS USADOS POR MIM:

git rebase

Git rebase a princípio parece fazer o mesmo que um comando git merge: ele integra dois branches em um branch único. Porém, esse comando refaz o histórico de commits, tornando-o linear. É o mais indicado para consolidar múltiplos branches.

Exemplo:
git rebase (base)

git pull –rebase

Essa é uma variação do comando pull mostrado anteriormente. A partir dessa instrução, o Git irá fazer um rebase (não um merge) depois de se utilizar um comando pull.

Exemplo:
git pull –rebase

git cherry-pick

Esse é um comando poderoso que permite selecionar qualquer commit específico de um brach e aplicá-lo a outro branch, sem precisar de uma mescla completa. A operação fica adicionada no histórico.

Exemplo:
$ git cherry-pick (commit-hash)

git archive

Esse comando Git combina múltiplos arquivos em um único arquivo, como se fosse um arquivo zipado. Esse pacote pode ser aberto depois e os arquivos contidos podem ser extraídos individualmente.

Exemplo:
$ git archive –format zip HEAD > archive-HEAD.zip

git blame

O comando “dedo-duro”, blame ajuda a determinar qual usuário realizou qual mudança em um determinado arquivo.

Exemplo:
$ git blame (nome_do_arquivo)

git tag

Tags são uma boa opção para marcar uma branch e evitar alteração, principalmente em releases públicos.

Exemplo:
$ git tag -a v1.0.0

git diff

Para comparar dois arquivos gits ou dois branches antes de passarem por um commit ou um push, é importante executar esse comando Git.

Exemplos:
comparando o repositório ativo com o repositório local: $ git diff HEAD comparando duas ramificações: $ git diff (branch de origem) (branch de destino)

git citool

Esse comando Git oferece uma alternativa gráfica ao commit.

Exemplo:
$ git citool
